# Packaging

Two distribution formats, per the plan. **Snap and Flatpak are intentionally
rejected** — their strict confinement blocks other processes' `/proc`, `fdinfo`
and device nodes, which would silently zero the GPU and cross-process collectors.

## `.deb` (primary)

Native Debian/Ubuntu package. `Depends:` are generated by `dh_shlibdeps` (never
hand-written), so the package pins exactly the Qt6 runtime it was built against.

```bash
# from the project root, on the TARGET Ubuntu release (e.g. 24.04):
sudo apt build-dep . 2>/dev/null || \
  sudo apt install build-essential cmake qt6-base-dev libdrm-dev debhelper
# debian/ lives under packaging/ in this repo; expose it at the root to build:
ln -sfn packaging/debian debian
dpkg-buildpackage -us -uc -b
```

For reproducibility, build in a clean chroot/container (sbuild/pbuilder) against
the target release. The CMake `install()` rules place the binary in `/usr/bin`,
the `.desktop` in `/usr/share/applications`, and the icon in
`/usr/share/icons/hicolor/scalable/apps`.

## AppImage (portable fallback)

```bash
./packaging/appimage/build-appimage.sh
```

Build on the **oldest** supported glibc/Ubuntu (e.g. 22.04) so the AppImage runs
there and on everything newer. The script uses `linuxdeploy` + its Qt plugin to
bundle the Qt platform plugin and platformtheme. Output lands in
`packaging/appimage/` (gitignored).

## Files

| Path | Purpose |
|---|---|
| `sysperf.desktop` | Shared desktop entry (installed by CMake; used by both formats). |
| `sysperf.svg` | Generic line-graph icon (no hardware/vendor references). |
| `debian/` | Debian packaging (`control`, `rules`, `changelog`, `copyright`, `source/format`). |
| `appimage/build-appimage.sh` | One-shot AppImage builder. |
